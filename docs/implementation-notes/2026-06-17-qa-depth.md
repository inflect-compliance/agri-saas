# 2026-06-17 — QA depth: regression-proofing the tier-1 farmer workflows

**Commit:** `feat/qa-depth` — ag E2E suite, seed fixture, k6 load tests, ag API contract snapshots, raised ag-usecase coverage.

## Design

Tier-1 farmer workflows carry regulatory + financial data (spray records,
stock ledger, yields, contracts). This adds five regression layers around
them — each independently locked by a CI gate:

```
 E2E (Playwright)      9× tests/e2e/ag-*.spec.ts ── real browser, isolated tenant per test
 Seed fixture          prisma/fixtures/ag-demo.ts ── 3 fields / 10 parcels / 5 products / 2 jobs
 Load (k6)             tests/load/ag-*.js ── p95<500ms budgets, gated in the load-smoke CI job
 Contracts (OpenAPI)   tests/contracts ── Location/OperationParcel/InventoryLot/YieldRecord/Contract
 Coverage              jest.thresholds.json ── usecases/ floor raised, locked in the ratchet
```

### Seed fixture — one shape, two runners

`seedAgDemo(ctx)` drives the REAL usecases (createLocation / createParcel
/ createItem / createLot / createFieldOperation), so seeded rows are
byte-identical to production-created data (parcels get `ST_Area` areaHa,
lots get a hash-chained RECEIPT, jobs get OperationParcel lines). It is the
in-process canonical builder — used by the integration test + available to
the dev seed.

Playwright can't cleanly import the `@/`-aliased app layer into the
browser-test runner (no e2e spec uses `@/`; importing the usecase graph
pulls instrumentation/env/encryption). So the ag E2E suite seeds the SAME
shapes through the authenticated HTTP API via `tests/e2e/ag-fixtures.ts` —
the HTTP twin of `ag-demo.ts`. Items + global Units (no create-API) are
seeded via raw Prisma, mirroring `offline-field-sync.spec.ts`.

### E2E — API-first, one robust UI check

Each spec seeds + mutates + asserts against SERVER TRUTH (the deterministic
path), then makes one light UI assertion (the entity name on its list
page). The financial path (`ag-spray-operation`) asserts the lot ledger
fell by Σ(dose×area) with one CONSUMPTION row per parcel and the job
auto-resolved — entirely via the API, so it can't flake on DOM timing.

Two environment facts shaped the specs:
- **No worker in E2E CI** (only `next start`). Only `spatial-import` is
  async (BullMQ); `ag-location-import` therefore asserts the synchronous
  parcel-authoring path + the spatial-import **202 enqueue** contract, not
  the worker-dependent parcel rows (those are covered by
  `spatial-import-hardening.test.ts`).
- **All modules on by default** (`resolveEnabledModules(null)` → ALL), so a
  fresh `/api/auth/register` tenant has JOURNAL+INVENTORY+GRAIN+PLANNING —
  the spray ledger fires and the inventory/grain/planning APIs don't 403
  without any module setup.

### Visual regression — structure, not pixels

The maplibre WebGL canvas is non-deterministic across GPUs/CI, so a pixel
snapshot would be perpetually flaky (and a committed baseline can't be
generated in a no-browser sandbox). `ag-map-visual` locks the deterministic
visual STRUCTURE of the spray-map page (the map region renders; exactly one
line per parcel; each with name + Done/Skip touch controls; Online status)
and attaches a full-page screenshot as a human-reviewable report artifact
(no pixel-diff assertion → no flake).

### Inventory pagination

`ag-inventory-pagination` needs a paginated endpoint. Added cursor
pagination to `GET /inventory/lots?limit=&cursor=` (dual-mode, mirroring
`/locations`): `listLotsPaginated` usecase + `InventoryRepository.listLotsPaginated`
(stable `(createdAt,id)` cursor, `limit+1` over-fetch). The bare GET still
returns the array (backward-compatible — the contract snapshot covers
both the `InventoryLot` item and the `InventoryLotPage`).

### Contracts

Two new DTOs — `OperationParcel` (raw line shape from `getFieldOperation`,
Decimals as strings) and `InventoryLot`/`InventoryLotPage` (mapped from
`listLots`) — plus wiring `location.dto` into `scripts/openapi-build.ts`
(it existed but was never registered). The existing
`tests/contracts/api-schemas.test.ts` then snapshots all five ag entities
and byte-compares the full spec against `public/openapi.json` — backward-
compat is enforced in CI for free.

### Coverage

`listLotsPaginated` is new usecase code only reachable via E2E/k6 (neither
counts toward Jest coverage), so the `ag-demo-fixture` integration test
covers it directly (cursor page + no-overlap), offsetting the dilution.
The `usecases/` floor is then raised within measured headroom
(67.78/65.55/77.99/76.32 → 67/63/77/75) and locked in the coverage ratchet.

## Decisions

- **Two seeders, not one.** The Playwright/app-layer import boundary forces
  an in-process builder (`ag-demo.ts`) + an HTTP twin (`ag-fixtures.ts`).
  They're cross-referenced in comments; the shape is identical.
- **Did NOT wire seedAgDemo into the dev seed.** A throw there breaks the
  whole seed → all E2E/load jobs. The integration test already exercises it
  loudly; the load tests pass against the existing seed (small data → p95
  trivially under budget). Not worth the blast radius.
- **`npm audit signatures`-style caution on coverage:** the coverage job is
  main-only (skips on PRs), so the raise is conservative + within already-
  measured headroom — the first post-merge main push won't break.
- **agro-signals spec drives data-streams**, the operator-facing agro
  surface, because AgroSignal rows are generated by the background
  weather-pull job with no synchronous HTTP read.
